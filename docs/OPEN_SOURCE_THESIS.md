# Open-source thesis

Agent tools are changing quickly. The durable layer should be the local agent environment, not the UI, model provider, or chat session.

AgentHome exists to make agent state portable:

- policies
- memories
- task history
- skills
- hooks
- model roles
- tool definitions
- workspace boundaries
- audit logs

The project should succeed even when users switch between multiple agent tools.

## What makes this contributable?

Useful contribution surfaces:

- provider adapters
- policy rules
- schema improvements
- workspace exporters
- audit viewers
- skill/recipe examples
- sandbox backends
- docs and examples

The project should avoid requiring contributors to understand a bespoke assistant personality or opaque workflow.
